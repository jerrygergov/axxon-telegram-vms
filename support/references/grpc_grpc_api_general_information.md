The [gRPC](https://en.wikipedia.org/wiki/GRPC) protocol is used to support the transport layer between _Axxon One_ services. 

The gRPC API allows to simplify the large-scale systems configuration.

The gRPC protocol uses special proto-files to generate the code base.

The proto-files contain some strict syntax that defines the interaction interface of the various program modules.

Attention!

To get _Axxon One_ proto-files, contact the [technical support service](https://help.axxonsoft.com/jira/servicedesk/customer/portal/3).

[Examples of gRPC API methods](/confluence/spaces/one20en/pages/246487067/Examples+of+gRPC+API+methods).

You can call the gRPC API methods using the HTTPS or HTTP requests (see [gRPC API method calls](/confluence/spaces/one20en/pages/246487045/gRPC+API+method+calls)) or through direct gRPC requests (see [Working with Axxon One via direct gRPC requests](/confluence/spaces/one20en/pages/246487104/Working+with+Axxon+One+via+direct+gRPC+requests)).

Attention!

To call the methods using the HTTPS requests, you should use the SSL certificate (see [Configuring the Web-Server](/confluence/spaces/one20en/pages/246484274/Configuring+the+Web-Server)) or disable the SSL verification on the side of the message sender.
